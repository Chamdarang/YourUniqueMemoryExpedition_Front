# Your Unique Memory Expedition(YUME) [Front]

## 프로젝트 개요
일본 여행 일정을 구글 스프레드시트로 관리하며 느꼈던 아카이빙의 한계와 불편함을 해소하기 위해 시작한 프로젝트입니다.  
단순한 일정 나열을 넘어, Google Maps API를 활용한 직관적인 동선 시각화와  
고슈인(御朱印) 및 기념품 수집 현황을 시각적으로 관리할 수 있는 인터페이스를 구현하고자 했습니다.  
사용자가 여행 전(PC)에는 계획을 세우고, 여행 중(Mobile)에는 신속하게 정보를 확인할 수 있는 상황별 최적화 UX를 목표로 했습니다.

## 개발 진행 방식
- AI를 활용한 UI/UX 구현  
  백엔드 개발자로서 익숙하지 않은 디자인 및 CSS 작업의 러닝 커브를 줄이기 위해 Gemini를 적극 활용했습니다.  
  기획한 와이어프레임과 의도를 AI에게 전달하여 빠르게 실제 코드로 변환함으로써, 디자인에 소요되는 시간을 최소화하며  
  이를 프로젝트에 맞게 적용하는 방식으로 프론트엔드 개발의 진입 장벽을 낮추고 개발 속도를 확보했습니다.


## 주요 중점
- 지도와 타임라인의 연동  
  현재 진행중인 일정과 이동경로(출발/도착)을 색상과 크기로 차별화하여  
  지도만 보고도 현재 상황을 직관적으로 인지할 수 있게 하고,     
  일정을 작성하는 순간에도 해당 위치와 경로를 지도에 즉시 표시하여  
  사용자가 전체적인 동선을 고려하며 효율적인 계획을 수립할 수 하고자 하였음.


- 사용자 상황에 맞춘 UX  
  여행의 단계에 따라 사용 기기가 달라지는 점에 착안하여 View를 최적화하여   
  PC View에서는 넓은 화면을 활용하여 전체 일정 편집, 장소 상세 관리, 드래그 앤 드롭 등 복잡한 조작 편의성에 집중했고,  
  Mobile View에서는 현장에서의 현재 위치 확인, 다음 일정 체크, 길 찾기 등 즉각적인 정보 확인에 최적화하고자 했음.

## 향후 계획
1. 수집품 관리 UX 고도화  
   개별 장소 상세 진입 없이 전체 수집 현황 파악을 가능하게 하려고 함   


2. 실시간 협업을 위한 웹소켓 클라이언트 구현  
   동행인과 여행 일정을 함께 확인하고 수정하기 위한 일정 공유 기능을 추가하려고 함 


3. 리팩토링 및 컴포넌트 분리
   빠른 구현을 위해 페이지 중심으로 로직을 통합하였으나,  
   향후 유지보수와 재사용성을 높이기 위해  
   비즈니스 로직(Hook)과 UI 컴포넌트를 분리하는 리팩토링을 진행할 예정
